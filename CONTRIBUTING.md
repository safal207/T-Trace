# Contributing

## How to contribute
- Open an issue describing bug, proposal, or spec clarification.
- Keep pull requests focused and reproducible.
- Add tests when changing validator or invariants.
- Update docs/spec when semantics change.

## PR checklist
- [ ] Validation script still passes on `examples/minimal.ttrace.jsonl`
- [ ] `pytest` passes
- [ ] Spec and schema stay aligned
- [ ] No generated artifacts or secrets committed
